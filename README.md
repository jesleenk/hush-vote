Hush Vote is Midnight private voting app.

Browser UI lives at repo root in Next.js. Compact contract lives in `contract/`. Vote flow uses connected Midnight wallet, indexer reads, and generated ZK assets copied into `public/zk/poll/`.

## What It Does

- Shows live ballot state from Midnight indexer
- Connects 1AM or Lace wallet in browser
- Lets user seal vote with wallet-signed proof flow
- Deploys new ballot from `/deploy`
- Keeps vote identity hidden from public tally

## Privacy Model

### What an observer can learn

- Ballot question and four answer options, because constructor text is disclosed.
- Current public tally, total vote count, poll state, and transaction timing through the indexer.
- That a vote transaction occurred and which vote circuit was called (`voteForA` through `voteForD`).

### What an observer cannot learn

- Voter identity from the Compact contract's public tally alone.
- Wallet private keys or the private witness data used to create the proof.
- Any personal information supplied outside this app and wallet flow.

Privacy scope: Hush Vote hides voter identity, not vote choice. Since each option has a separate public circuit and tally, observers can see which option's counter changed. Users should not submit personal information as ballot text.

## How It Works

1. `contract/src/poll.compact` defines ballot storage and vote circuits.
2. `contract build` compiles contract into `contract/src/managed/poll/`.
3. `contract/package.json` copies generated `keys/` and `zkir/` into `public/zk/poll/`.
4. `lib/poll-contract.ts` loads compiled contract and generated assets.
5. `lib/poll-indexer.ts` reads public ballot state directly from browser.
6. `lib/midnight-browser.ts` connects browser wallet and creates proof/provider session.
7. `app/components/poll-card.tsx` renders ballot UI and submits vote from browser wallet.
8. `app/deploy/deploy-client.tsx` deploys new ballot contract from wallet.

## Contract Details

Contract file:

- [`contract/src/poll.compact`](./contract/src/poll.compact)

State:

- `state`: `OPEN` or `CLOSED`
- `question`: ballot question
- `optionA` to `optionD`: 4 answers
- `votesA` to `votesD`: per-choice counters
- `totalVotes`: total vote counter

Circuits:

- `voteForA()`
- `voteForB()`
- `voteForC()`
- `voteForD()`

Constructor:

- Takes 1 question and 4 options
- Discloses ballot text into ledger state
- Starts ballot in `OPEN`

Important note:

- Contract is fixed to 4 options
- UI and deploy page both assume 4 answers
- No server-side vote submitter exists

## Runtime Config

`lib/poll-config.ts` reads these env vars:

- `NEXT_PUBLIC_MIDNIGHT_NETWORK`
- `NEXT_PUBLIC_CONTRACT_ADDRESS`
- `NEXT_PUBLIC_MIDNIGHT_INDEXER_URL`
- `NEXT_PUBLIC_MIDNIGHT_PROOF_SERVER_URL`

Current defaults:

- Network: `preprod`
- Indexer: `https://indexer.preprod.midnight.network/api/v4/graphql`
- Proof server: `http://localhost:6300`
- Contract address: preprod ballot address in `.env.example`

`.env.example`:

```env
NEXT_PUBLIC_MIDNIGHT_NETWORK=preprod
NEXT_PUBLIC_CONTRACT_ADDRESS=5f5c3db9eba1bf5b7a64273ef9fffad5cc8754c323d233730a75126fd1df2949
NEXT_PUBLIC_MIDNIGHT_INDEXER_URL=https://indexer.preprod.midnight.network/api/v4/graphql
NEXT_PUBLIC_MIDNIGHT_PROOF_SERVER_URL=http://localhost:6300
```

## Setup

1. Install Node.js 22 or newer.
2. Install `npm` deps:

```bash
npm install
```

3. Install Compact compiler if you want to rebuild contract:

```bash
curl --proto '=https' --tlsv1.2 -LsSf https://github.com/midnightntwrk/compact/releases/latest/download/compact-installer.sh | sh
```

4. Build contract assets:

```bash
npm run contract:build
```

5. Start app:

```bash
npm run dev
```

6. Open app and connect 1AM or Lace wallet.
7. Switch wallet to Midnight `preprod`.

## How To Use

### Vote

1. Open home page.
2. Connect wallet.
3. Pick 1 answer.
4. Seal vote.
5. Wait for indexer refresh to show updated tally.

### Deploy New Ballot

1. Open `/deploy`.
2. Connect wallet.
3. Edit question and 4 options.
4. Click `Deploy survey`.
5. Copy deployed contract address.
6. Update `NEXT_PUBLIC_CONTRACT_ADDRESS` if you want home page to read new ballot.

### Local Proof Server

If wallet flow needs local proof server, run one at `http://localhost:6300` and set:

```env
NEXT_PUBLIC_MIDNIGHT_PROOF_SERVER_URL=http://localhost:6300
```

If you want another indexer, set:

```env
NEXT_PUBLIC_MIDNIGHT_INDEXER_URL=https://indexer.preprod.midnight.network/api/v4/graphql
```

## Troubleshooting

- Wallet not found: install 1AM or Lace, then reload.
- Wrong network: switch wallet to Midnight `preprod`.
- Ballot empty: set `NEXT_PUBLIC_CONTRACT_ADDRESS` to deployed contract.
- Proof fail: make sure `public/zk/poll/` exists and wallet can reach proof server.
- Tally stale: indexer may need a few refresh cycles after vote.

## Notes

- `app/api/poll/vote/route.ts` stays as server-side fallback; vote submit happens in browser wallet.
- `app/page.tsx` anchors ballot section as `#poll`.
- Generated contract output under `contract/src/managed/poll/` should not be edited by hand.
- Public ballot reads happen in browser via `lib/poll-indexer.ts`.
